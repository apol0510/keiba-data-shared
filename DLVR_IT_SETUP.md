# dlvr.it連携設定ガイド

## 📋 概要

RSS feedを使ってX（旧Twitter）に自動投稿する設定方法。
X API直接投稿よりも安定しており、X側の障害の影響を受けにくい。

## 🎯 フロー（レース毎）

```
管理画面で第11R保存
  ↓ 即座
keiba-data-sharedに自動コミット・プッシュ
  ↓ 5-10分
Netlify自動デプロイ
  ↓ 即座
RSSフィード自動更新（第11Rが追加）
https://data.keiba-intelligence.jp/rss.xml
  ↓ 最大15分
dlvr.it が定期チェック
  ↓ 即座
新しいレースを検出
  ↓ 即座
Xに第11R自動投稿 🎉

管理画面で第12R保存
  ↓ 同じフロー
Xに第12R自動投稿 🎉
```

## 🚀 RSS Feed URL

**メインRSS:**
```
https://data.keiba-intelligence.jp/rss.xml
```

**特徴:**
- 南関競馬・JRA競馬の結果を**1レース毎**に配信 🆕
- 直近7日分のデータを含む（約84件）
- 最大100件のアイテム
- 日付・レース番号順（新しい順、同日内は12R→1R）
- 絵文字付きタイトル
- **1着馬・騎手・三連単配当を含む** 🆕

## 🔧 dlvr.it 設定手順

### 1. アカウント作成

1. [dlvr.it](https://dlvr.it/) にアクセス
2. 「Sign Up」をクリック
3. メールアドレス・パスワードを設定
4. 無料プラン（Free Plan）を選択

### 2. ソース（RSS Feed）追加

1. ダッシュボードで「Add Source」をクリック
2. 「RSS Feed」を選択
3. RSS Feed URLを入力:
   ```
   https://data.keiba-intelligence.jp/rss.xml
   ```
4. 「Check Feed」で接続確認
5. 「Save」をクリック

### 3. ルート（Route）設定

#### 3-1. X（Twitter）連携

1. 「Add Route」をクリック
2. 「Twitter」を選択
3. Xアカウントで認証（OAuth）
4. 権限を許可

#### 3-2. 投稿フォーマット設定

**推奨設定:**

| 項目 | 設定値 |
|------|--------|
| **Title** | `{title}` |
| **Description** | `{description}` |
| **Link** | `{link}` |
| **投稿形式** | `{title}\n\n{description}\n\n{link}\n\n#南関競馬 #JRA #競馬結果` |

**投稿例（レース毎）:**
```
🐴 2026/03/04 川崎競馬 第11R ブルートシュタインチャレンジ

1着 7番マキシマムパワー（町田直希）
三連単 7-9-11 72,850円

詳細・払戻金はこちら

https://data.keiba-intelligence.jp/nankan/results/2026/03/04/kawasaki/11/

#南関競馬 #JRA #競馬結果
```

#### 3-3. チェック頻度設定

| 項目 | 推奨値 |
|------|--------|
| **Check Frequency** | 15分ごと |
| **Max Posts Per Check** | 5件 |
| **Post Immediately** | ON |

### 4. フィルター設定（オプション）

特定のキーワードで投稿を制御できます：

**例: 南関競馬のみ投稿する場合**
1. 「Filters」タブをクリック
2. 「Include only items that contain」を選択
3. キーワード: `南関競馬`

**例: 特定の競馬場のみ投稿する場合**
1. キーワード: `大井競馬` または `東京競馬`

### 5. 動作確認

1. 「Test Route」をクリック
2. 最新のRSSアイテムが表示される
3. 「Post to Twitter」で手動投稿テスト
4. Xアカウントで投稿を確認

## ⚙️ 設定のコツ

### 1. 重複投稿を防ぐ

dlvr.itは既に投稿したアイテムをトラッキングします。
- RSSのguidフィールドを使用
- 同じguidは二度と投稿されない

### 2. 投稿タイミング

**無料プラン:**
- チェック頻度: 最小15分
- 最大投稿数: 1日100件

**有料プラン:**
- チェック頻度: 最小5分
- 最大投稿数: 無制限

### 3. エラーハンドリング

dlvr.itは以下の場合に自動的にリトライします：
- X側のAPI制限
- ネットワークエラー
- 一時的な障害

## 🔍 トラブルシューティング

### RSS Feedが更新されない

**原因:** Netlifyのビルドが完了していない

**解決策:**
1. [Netlify管理画面](https://app.netlify.com/) でビルドログを確認
2. ビルド完了まで5-10分待つ
3. https://data.keiba-intelligence.jp/rss.xml にアクセスして確認

### dlvr.itが新しい記事を検出しない

**原因:** チェック頻度の設定

**解決策:**
1. dlvr.itダッシュボードで「Check Now」を手動実行
2. チェック頻度を確認（最小15分）
3. RSS FeedのpubDateを確認

### 投稿がXに表示されない

**原因:** X側のAPI制限またはシャドウバン

**解決策:**
1. dlvr.itのログを確認
2. Xアカウントで投稿制限がないか確認
3. 投稿頻度を下げる（Max Posts Per Checkを減らす）

## 🚨 X API直接投稿との比較

| 項目 | X API直接投稿 | dlvr.it経由投稿 |
|------|--------------|----------------|
| **安定性** | ❌ 不安定（X側の障害で停止） | ✅ 安定（RSS経由） |
| **認証** | OAuth 1.0a（複雑） | OAuth 2.0（簡単） |
| **エラーハンドリング** | 自前実装が必要 | 自動リトライ |
| **コスト** | 無料（API制限あり） | 無料プランあり |
| **投稿タイミング** | 即座 | 15分遅延（無料プラン） |
| **管理** | コード修正が必要 | Webダッシュボードで設定 |

## 📊 メリット・デメリット

### メリット

✅ **安定性**: X APIの障害に影響されない
✅ **簡単**: コード不要、Webダッシュボードで設定
✅ **自動リトライ**: エラー時も自動的にリトライ
✅ **柔軟**: フォーマットを自由にカスタマイズ可能
✅ **管理が楽**: Netlify環境変数の設定不要

### デメリット

⚠️ **遅延**: 15分ごとのチェック（無料プラン）
⚠️ **外部依存**: dlvr.itのサービスに依存
⚠️ **制限**: 無料プランは1日100件まで

## 🎉 まとめ

**推奨設定:**
1. RSS Feed: `https://data.keiba-intelligence.jp/rss.xml`
2. チェック頻度: 15分ごと
3. Max Posts: 5-10件/チェック（レース毎配信のため）
4. フィルター: 必要に応じて設定

**運用フロー（レース毎）:**
```
管理画面で各レース保存 → RSS自動更新 → dlvr.itが検出 → X自動投稿 🎉
```

**タイムライン:**
- 管理画面で保存: レース確定後すぐ
- Netlifyデプロイ: 5-10分
- dlvr.itチェック: 最大15分
- **合計: 保存から20-25分後に自動投稿** ⏱️

**投稿頻度:**
- 1日12R × 1競馬場 = 12投稿/日
- 複数競馬場開催時は投稿数増加
- dlvr.it無料プラン: 1日100件まで（十分対応可能）
